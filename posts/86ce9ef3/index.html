mermaid快速入门

什么是Mermaid？

  • Mermaid 是一款基于 JavaScript 的绘图和制图工具，它使用类似 Markdown 的文本定义和一个渲染器来创建和修改复杂图表。Mermaid 本质是 “用文字写图表的语法”，但它需要“渲染工具” 把文字转换成图表（就像 Markdown 需要编辑器渲染成排版好的文字）。
  • 项目地址：https://github.com/mermaid-js/mermaid
  • Mermaid Live 编辑器中提供了图表示例，它也是一个很好的练习场所。

怎么使用？

  • 使用特定的Mermaid渲染器；
  • 使用集成了Mermaid渲染功能的Markdown编辑器，如Typora。使用时，需要将代码块的语言选择为Mermaid。

功能

绘制：

  • 饼状图：使用pie关键字，具体用法后文将详细介绍
  • 流程图：使用graph关键字，具体用法后文将详细介绍
  • 序列图：使用sequenceDiagram关键字
  • 甘特图：使用gantt关键字
  • 类图：使用classDiagram关键字
  • 状态图：使用stateDiagram关键字
  • 用户旅程图：使用journey关键字

语法结构

所有图表定义都以图表类型声明开头，随后是图表及其内容的定义。此声明告知解析器代码应生成哪种类型的图表。唯一的例外是Frontmatter配置。

行注释可以忽略%% 之后所有的内容。

未知单词和拼写错误会导致图表出错，而参数则会失效。

流程图（FlowChart）

https://mermaid.js.org/syntax/flowchart.html

流程图由节点（几何形状）和边（箭头或线条）组成。

声明

声明：flowchart LR:流程图类型+布局方向。

  • LR：流程图的节点从左到右依次排列（水平方向）。
  • TB/TD：Top to Bottom（从上到下，默认方向）
  • RL：Right to Left（从右到左）
  • BT：Bottom to Top（从下到上）

节点

Warning：

如果在流程图节点中使用单词“end”，需要将整个单词或其中任何一个字母大写（例如，“End”或“END”），或者使用此解决方法。全部小写字母输入“end”会导致流程图出错。

flowchart LR
	id

带文本的节点

可以在框中设置与节点 ID 不同的文本。如果多次设置文本，则会使用节点最后一次显示的文本。

是否使用双引号包裹节点文本，以及由此带来的文本解析规则差异：


  • 直接文本

  • Unicode文本

  • Markdown格式文本

直接文本

flowchart LR
    id1[This is the text in the box]

  • 文本中不能包含特殊符号（如 [ ]、{ }、( )、|、-> 等），否则会被解析为语法符号，导致渲染失败。

  • 文本中的空格、字母、数字可正常显示。

  • 适合纯英文 / 纯中文、无特殊符号的简单文本。

Unicode文本

使用""包裹Unicode文本。

flowchart LR
    id["This ❤ Unicode"]
  • 支持Unicode 字符（如 emoji、特殊符号、中文标点、日文 / 韩文等）。
  • 文本中可以包含特殊符号（如 [ ]、{ }、( )、|、-> 等），双引号会将其视为普通文本，不参与语法解析。
  • 适合包含 emoji、特殊符号、多语言的复杂文本。

Markdown格式文本

使用双引号和反引号"``"将文本括起来

flowchart LR
    markdown["`This **is** _Markdown_`"]
    newLines["`Line1
    Line 2
    Line 3`"]
    markdown --> newLines

Mermaid 流程图中的特殊形状（v11.3.0+）

图表（Icon）

flowchart TD
	A@{ icon: "fa:user", form: "square", label: "User Icon", pos: "t", h: 60 }
  • icon：来自已经注册的图标包
  • form：icon背景的形状，没有定义则没有背景。可选项：
    • square
    • circle
    • rounded
  • label：与图标关联的文本标签
  • pos：标签的位置
    • t：top顶部
    • b：buttom
  • h：图标的高度

图像

flowchart TD
	A@{img: "https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRDZ81HSoVvOR2tNuDXMP3TEBPU5NBkUY4usw&s", label: "邓紫棋", pos: "t", w: , h: , constraint: "on"}
  • constraint：是否限制图像节点的大小（宽高比）

隐性Link

只调整节点的布局位置，但不显示任何连接线。

  1. 强制节点布局
    • 在流程图里，节点默认会按代码顺序和布局方向（如 LR、TB）自动排列。
    • 用隐形链接可以打破这个默认规则，强制让两个节点保持在同一行或同一列，同时又不显示连线。
  2. 调整节点间距
    • 当你想让某些节点靠得更近或更远，又不想用可见的连接线时，它是个非常灵活的工具。
  3. 复杂图表排版
    • 在多分支、多层级的复杂流程图中，隐形链接可以用来对齐节点，让整体布局更整齐、逻辑更清晰。
flowchart LR
	A~~~B

链式Link

flowchart LR
	A -- text --> B -- text2 --> C
flowchart LR
	a --> b & c -->d
flowchart TB
	A & B --> C & D

为Link赋ID

在边的语法前加上该ID+@，定义了边的属性：精准定制样式（颜色、粗细、虚线等）、添加动态动画（流动效果、速度控制）、绑定交互事件（点击跳转、触发动作）、提升图表可维护性（复杂场景下清晰区分每条边）

flowchart LR
	A e1@--> B;

开启动画

flowchart LR
	A e1@==> B
	e1@{animate: true}

选择动画类型

flowchart LR
	A e1@--> B
	e1@{animation: slow}
  • animation：动画速度
    • fast
    • slow

使用classDef语句实现动画

可以通过给边分配一个类，在classDef语句中定义动画属性来为边添加动画效果

flowchart LR
	A e1@--> B
	classDef type1 stroke-dasharray: 9,5,stroke-dashoffset: 900,animation: dash 25s linear infinite;
	class e1 type1
  • classDef type1：定义一个带有样式和动画属性的名为type1类
  • class e1 type1将类应用于边e1

Link长度

流程图中的每个节点最终都会根据其连接的节点，被分配到渲染后的图形中的一个层级，即垂直或水平层级（取决于流程图的方向）。默认情况下，链接可以跨越任意数量的层级，但可以通过在链接定义中添加额外的短横线来指定某个链接的长度。

通过增加额外的破折号，增加跨越的层级，（当链接标签写在链接中间时，多余的短横线必须添加到链接的右侧）：

flowchart TD
	A[start] --> B{Is it?}
	B -->|Yes| C[OK]
	C --> D[Rethink]
	D --> B
	B ---->|No| E[End]
长度 1 2 3
普通的 --- ---- -----
普通箭头 --> ---> ---->
厚的 === ==== =====
箭矢密布 ==> ===> ====>
点状 -.- -..- -...-
带箭头的点 -.-> -..-> -...->

流程图

使用 graphtype 流程图，还可以设置子图之间的边

flowchart TB
    c1-->a2
    subgraph one
    a1-->a2
    end
    subgraph two
    b1-->b2
    end
    subgraph three
    c1-->c2
    end
    one --> two
    three --> two
    two --> c2

子图中的方向

使用 graphtype 流程图，可以使用 direction 语句来设置子图的渲染方向

flowchart LR
	subgraph TOP
		direction TB
		subgraph B1
			direction RL
			i1 --> f1
		end
		subgraph B2
			direction BT
			i2 --> f2
		end
	end
	A --> TOP --> B
	B1 --> B2

局限

如果子图的任何节点连接到外部，则子图的方向将被忽略，子图将继承父图的方向。

flowchart LR
    subgraph subgraph1
        direction TB
        top1[top] --> bottom1[bottom]
    end
    subgraph subgraph2
        direction TB
        top2[top] --> bottom2[bottom]
    end
    %% ^ These subgraphs are identical, except for the links to them:

    %% Link *to* subgraph1: subgraph1 direction is maintained
    outside --> subgraph1
    %% Link *within* subgraph2:
    %% subgraph2 inherits the direction of the top-level graph (LR)
    outside ---> top2

相互作用

可以将点击事件绑定到节点，点击可以触发 JavaScript 回调，也可以触发在新浏览器标签页中打开的链接。

click语法

  1. 点击节点 -> 执行JS函数

    • A：节点 ID
    • callback：JS 函数名
    • 引号里：鼠标悬停时显示的提示文字

    JS回调函数：

    点击后会弹出提示框的函数。

  2. 点击节点 -> 打开链接

    • B：节点
    • 链接：要跳转的地址
    • 引号：提示文字
    • _blank：在新标签页打开
      • _self = 在当前页打开（默认）
      • _parent / _top = 框架页打开（很少用）

嵌入到HTML中（前端网站）

  • 使用时此功能被禁用 securityLevel='strict'

  • 使用时此功能被启用 securityLevel='loose'

在Typora等编辑器中

不需要写JS代码，只需要写Mermaid语法，编辑器会自动处理

flowchart LR
    A[首页] --> B[文章]
    B --> C[评论]
    C --> D[点赞]

    %% 点击 A → 打开 GitHub（新标签）
    click A "https://github.com" "打开 GitHub" _blank

    %% 点击 B → 打开百度（新标签）
    click B "https://www.baidu.com" "打开百度" _blank

    %% 点击 C → 打开 Google（新标签）
    click C "https://www.google.com" "打开 Google" _blank

    %% 点击 D → 打开 Typora 官网（新标签）
    click D "https://typora.io" "打开 Typora 官网" _blank
